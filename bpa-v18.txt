bpa-v18.txt — “Bitter-Lesson Bit Allocation: Adam v̂ as KV Precision Signal”

Mission:
v16 showed manual sensitivity-based INT4/INT8 allocation works.
v17 proposed FIM-guided allocation.
v18 formalizes the Bitter-Lesson version:

Primary signal = Adam second moment (v̂).
Stabilized via 4th-root transform (Bitter7 logic).
Full Fisher only as ablation.

We are not chasing theoretical Fisher.
We are exploiting optimizer statistics already available.

Core hypothesis:
Layer-wise Adam v̂ (aggregated over KV-related parameters) predicts which layers require higher KV precision.

If true:
- Automated allocation replaces manual S2.
- No expensive empirical sweeps needed.
- Scales to larger models.

========================================================
GLOBAL SETTINGS
========================================================
Quality tolerance:
- tol=3% strict PASS across seeds {0,1,2}
- Report tol=1% as bonus.

Contexts:
- L ∈ {8192, 16384, 32768}
- Regimes: r1 batch=1, r2 microbatch=4
- Decode_steps >= 256

KV storage:
- Must use real int4/int8 packed storage.
- No simulated casts allowed in headline.

========================================================
PHASE 0 — Baseline Lock (1h)
========================================================
Re-run:
- dense
- INT8 everywhere
- S2 manual schedule (v16 best)

Confirm reproducibility.

========================================================
PHASE 1 — Adam v̂ Sensitivity Extraction (2–4h)
========================================================
Goal:
Compute per-layer sensitivity score using optimizer state.

If model was trained with Adam and v̂ is available:
  Extract v̂ tensors.
Else:
  Run short calibration fine-tune loop (few hundred steps) with Adam to build v̂.

For each layer ℓ:
  Identify parameters contributing to KV projections (W_k, W_v).
  Compute:

    raw_score[ℓ] = mean(v̂ for KV-related params)

Now apply Bitter7 stabilization:

    score[ℓ] = (raw_score[ℓ])^(1/4)

Rationale:
- v̂ is heavy-tailed.
- 4th-root compresses scale while preserving ranking.
- Matches prior pruning logic.

Normalize:
  score_norm[ℓ] = score[ℓ] / max(score)

Deliver:
- artifacts/sensitivity/adam_vhat_raw.json
- artifacts/sensitivity/adam_vhat_root4.json

========================================================
PHASE 2 — Allocation Policy (Greedy + Budget) (2–4h)
========================================================
Define bit options:
- INT4
- INT8

Define byte cost per layer:
  cost_INT4[ℓ]
  cost_INT8[ℓ]

Start with all INT4.
Sort layers by score_norm descending.

Greedy allocation:
  Upgrade highest-score layers to INT8
  until quality constraint satisfied.

Procedure:
  For k = 0 to n_layers:
    Set top-k layers to INT8.
    Evaluate quickly at L=8192.
    If PASS @3% across seeds:
        record minimal k.
        break.

Then:
  Validate chosen k at L={16384,32768}, r1+r2.

Compare against:
- Manual S2
- Random schedule (same k)
- Empirical sensitivity schedule (oracle from v16)

Success criteria:
- Adam-root4 schedule matches or beats S2 kv_bytes_ratio.
- Random schedule fails at same k.
- Adam ranking outperforms random consistently.

========================================================
PHASE 3 — Correlation Analysis (Sanity, not obsession) (1–2h)
========================================================
Compute:
- Spearman correlation between:
    empirical sensitivity (v16)
    raw v̂ ranking
    4th-root v̂ ranking

Report:
- Which correlates best with empirical INT4 degradation.
- Whether root4 improves ranking stability.

We are not optimizing Fisher.
We are validating Adam as sufficient signal.

========================================================
PHASE 4 — 7B / 8B Replication (3–6h)
========================================================
Target:
- Llama-3-8B preferred.
- If not feasible, use strongest available 7B.

Steps:
1) Extract v̂ from optimizer state or build via short calibration.
2) Generate Adam-root4 schedule.
3) Evaluate at L={8192,16384} (32768 if feasible).
4) Compare against:
    INT8 everywhere
    manual transfer of small-model schedule
    random control

Success:
- >=40% KV byte reduction vs INT8 baseline
- PASS @3%

If W7900 memory insufficient:
  Flag: “H100 required for A2 replication”
  Continue A1/A3 on current model.

========================================================
PHASE 5 — Bandwidth-Bound Regime (2–4h)
========================================================
Increase:
- microbatch to 8
- longer decode
- 8B model if available

Measure:
- attention kernel time %
- memory read volume
- latency delta vs INT8 baseline

Goal:
Show kv_bytes_ratio reduction produces measurable latency improvement
in at least one regime.

If compute-bound:
  Document clearly.

========================================================
PHASE 6 — Optional Full Fisher Ablation (Time Permitting)
========================================================
Compute diagonal Fisher via E[g^2] over calibration set.

Repeat allocation using Fisher scores (no root4).

Compare:
- Fisher schedule
- Adam-root4 schedule

If Fisher marginally better but far more expensive:
  Conclude Adam-root4 is Bitter-optimal.

========================================================
HEADLINE POSSIBILITIES
========================================================
CASE A:
Adam-root4 matches S2 with equal or fewer INT8 layers.
→ Automated precision allocation achieved.

CASE B:
Adam-root4 slightly worse than empirical but far cheaper.
→ Accept as scalable solution.

CASE C:
Adam signal weak.
→ Need KV-logit-level sensitivity signal instead.

CASE D:
8B replication confirms scaling.
→ Strong generalization claim.

STOP CONDITIONS
- No manual tweaking after schedule generated.
- No cherry-picked seeds.
- No simulated KV storage in headline.
- No claiming Fisher equivalence without correlation evidence.

Time budget:
Designed for 12h including calibration, sweeps, and replication.
If time remains:
  Extend to head-level v̂ aggregation.
  Test continuous bit allocation (INT4/6/8 if supported).

EXECUTION LOG

Phase 1 (DONE, commit ca936cf):
  200-step Adam calibration on 48 KV params. Raw v-hat range
  2.8e-7 to 4.1e-4 (1467x). Layer 0 dominant (42x above median).
  Root4 compresses to 6.2x. Ranking: [0,1,4,3,2,16,...].
  Saved to artifacts/sensitivity/.

Phase 2 (DONE, commit 75f5249):
  Greedy search found k=2 (layers {0,1}) for PASS@3% at L=8K.
  adam@8K: +1.25%, adam@16K: +4.99% FAIL, adam@32K: +2.24%.
  random: catastrophic (21-430%). S2 k=6: FULL PASS@1%.
  Adam layer 1 replaces empirical layer 8 -- this costs ~3% at 16K.

Phase 3 (DONE, commit 53bb4da):
  Spearman rho=0.40, p=0.052. Moderate, borderline significant.
  Root4 does not change ranking. Layer 8 misranked (#2 empirical
  vs #13 adam). Adam captures parameter curvature, not activation
  sensitivity.

Phase 4 (DONE, commit 1369c5a):
  Qwen2.5-7B-Instruct on W7900 FP16. 100-step calibration.
  Minimal k=6 (22xINT4 + 6xINT8). adam@8K: +1.40% PASS,
  adam@16K: +5.96% FAIL. Random@8K: +1.38% (nearly identical!).
  At k=6/28, budget is generous enough to mask bad choices.
  CASE B confirmed for 7B: signal too weak to differentiate.

Phase 5 (DONE, commit 046cc11):
  W7900 compute-bound. <0.3% latency delta across all regimes
  (mb=1/4/8, L=8K/16K/32K). No decode speed benefit from KV
  compression on this hardware. Memory capacity benefit only.

Phase 6 (SKIPPED):
  Fisher ablation not run. Given weak adam signal, Fisher would
  need to operate at activation level to improve, which is a
  different approach entirely.

FINAL VERDICT: CASE B
  Adam v-hat is cheap and better than random at tight budgets
  (k=2: adam +1.25% vs random +21%). But inferior to empirical
  S2 (+0.30%) and indistinguishable from random at generous
  budgets on 7B. Not a replacement, but a viable first-pass
  heuristic.
