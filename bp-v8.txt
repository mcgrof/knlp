# Phase 2: Adversarial “entropy shift” stress tests (sequence changes)
# =============================================================================
# v7 indicates RA behaves like a slow, sequence-level persistence signal (shuffled not hurting).
# Now we must create cases where slow persistence is WRONG.

# Implement stress generators that keep token distribution similar but change dependency structure:
# 2.1 stress_topic_switch:
# - First half is from one document/topic, second half from a different document/topic.
# - Evaluate at positions AFTER the switch: the “useful past” should reset.
#
# 2.2 stress_late_binding:
# - Plant a key definition/fact late (near t ~ 0.75L) that is referenced immediately afterwards.
# - If gate keeps attending early chunks due to persistence, it should fail here.
#
# 2.3 stress_pointer_task (synthetic):
# - Insert a random key-value table early, query key late; then reverse (table late, query early).
# - This forces the gate to detect “needed far context” precisely.
#
# For each stress test:
# - compare dense vs BPA variants at matched average KV_kept.
# - measure PPL delta specifically on the “critical region” tokens (not only whole-sequence average).
#
# Required outputs:
# - PPL_by_region: early / pre-switch / post-switch / critical-window
# - KV_selection_heatmap over time: fraction of selected far chunks vs token position t.

# Stop rule Phase 2:
# - If BPA fails badly only on late-binding/pointer tasks, that confirms “slow signal” limitation.
#   Then v9 should add a fast conditional feature (local uncertainty/disagreement) to complement RA.
