bpa-v21.txt — “Exploit g=4, beat kv_ratio=0.33, and see if k<4 becomes viable (with real byte accounting)”

Context (what v20 established):
- Baselines: dense PASS; INT8 PASS; S2_manual_k6 PASS with kv_ratio≈0.333. :contentReference[oaicite:1]{index=1}
- Oracle k-sweep (INT4 g=32): k*=4 PASS@3% across all L; k=2 fails at 16K (+5.6%). :contentReference[oaicite:2]{index=2}
- Theory predicts k*=5 vs observed 4 (within 1 layer), supporting an architectural floor. :contentReference[oaicite:3]{index=3}
- V-only INT8 is a DEAD END (K quant dominates; RoPE damage).
- Rescaling is a DEAD END; norm clamp is marginal.
- Tight-group INT4 (g=4) is BEST: all-INT4 goes from +429%→+4.46% at 32K; k=4 max delta improves to +1.70%.
- v21 branch target: combine g=4 with k=2–4 and compute TRUE kv_ratio including scale overhead. :contentReference[oaicite:7]{index=7}

Mission for v21:
A) Compute TRUE KV cache byte cost for each quant scheme (including scale/zero metadata).
B) Use g=4 where it matters, and search for the best (k, g) combo that beats S2_k6’s kv_ratio≈0.333 while maintaining PASS@3% at L up to 32K.
C) Determine if the k-floor is truly “k>=4” or if reducing σ₄ with g=4 lets k=2–3 pass at all L.
D) If k<4 still fails, quantify exactly why (which layer/time positions cause failure), and propose the minimum additional knob needed (likely: K-specific treatment or learned scale calibration at top layers).

Non-negotiables:
- Model: same “real model of value” as v20.
- L ∈ {8192, 16384, 32768}, plus L=4096 sanity.
- Seeds {0,1,2}. Decode_steps >= 256.
- Real packed KV storage for headline runs (no simulated casts).
- PASS criterion: tol=3% across all seeds and all L; report tol=1% too.
- Always include baselines: dense, INT8-all, S2_manual_k6.

Deliverables:
- artifacts/v21/kv_bytes_accounting.md + kv_bytes_accounting.json
- artifacts/v21/search_grid.json (all tried configs)
- bpa_v21_scoreboard.json
- bpa_v21_final_report.md
- bpa_v21_branch_tree.md

Output:
- artifacts/v21/failure_attribution.json
- Include a short summary in report explaining why k<4 still fails even with g=4.

========================================================
PHASE 4 — Minimal Next Knob (only if needed) (2–6h)
========================================================
If (g=4, k=3) is “close” (e.g., max_delta 3.0–3.8%):
Try ONE additional minimal intervention, chosen by evidence:

Option 1 (preferred): K-specific treatment for sensitive layers
- For the top 1–2 sensitive layers, keep K at INT8 and allow V at INT4 g=4 (or INT8 if needed).
- Rationale: v20 proved K/RoPE is the fragility.
- This is NOT V-only INT8; it is “K-only INT8” where needed.

Option 2: Learned scale calibration ONLY for the top sensitive layers
- Freeze model weights.
- Learn INT4 scales (and optionally zero points) for layers {0,8,1,2} on calibration set.
- Objective: minimize Δlogit KL or ΔNLL.
- Keep steps <= 2k and report improvement.
- This targets reducing sigma4 at the few layers that dominate noise. :contentReference[oaicite:11]{index=11}

Do NOT do:
- post-quant rescaling (known dead end)
- norm clamp beyond a quick check (marginal)
