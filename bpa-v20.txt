bpa-v20.txt — “Break the k-floor: Theory + 3 Attack Paths (A/B/C) with Unified Harness”

Mission:
v19 proved k has a floor: even oracle k=2 fails at long context, so the constraint is accumulation.
v20 must:
(1) formalize accumulation mathematically and fit a bound on k,
(2) implement and evaluate interventions that reduce accumulation and/or per-layer noise,
across three paths:
A) reduce accumulation,
B) reduce damage per layer,
C) structural modifications.

Non-negotiables:
- Model: same “real model of value”.
- L ∈ {8192, 16384, 32768}, plus 4096 sanity.
- Seeds {0,1,2}. Decode_steps >= 256.
- Real packed KV storage for headline runs.
- Baselines always included: dense, INT8-all, S2_manual (k=6 best), oracle schedule if available.

Outputs:
- bpa_v20_final_report.md
- bpa_v20_scoreboard.json
- bpa_v20_branch_tree.md
- artifacts/v20/theory_k_floor.md + fitted parameters
- artifacts/v20/k_sweeps/*.json (quality vs k curves)
- code changes + logs

========================================================
PHASE 1 — Theory: accumulation model + lower bound on k (2–4h)
========================================================
Implement a small theory module:
- Model residual recursion with additive noise per layer.
- Derive bound: k >= (D*sigma4 - B)/(sigma4 - sigma8) under gamma≈1.
- Generalize to per-layer sigmaℓ with weights wℓ (amplification).
- Fit sigma4, sigma8, B (and optionally gamma) from the observed k-sweep curve.

How to estimate sigmaℓ empirically:
Option 1: use oracle ΔPPL per-layer INT4 ablation as sigmaℓ proxy.
Option 2: use residual_ratio C3 as sigmaℓ proxy (scaled).
Option 3: directly measure ||Δresidual|| under quant at layer ℓ.

Deliver:
- artifacts/v20/theory_k_floor.md with derivation
- artifacts/v20/theory_fit.json with fitted params
- show predicted k* vs observed k*

Success:
- The fitted model predicts k* within ±1 layer.

========================================================
PHASE 2 — Path A: Reduce accumulation (3–6h)
========================================================
A1) Per-head precision (within sensitive layers)
- Implement per-head INT8 selection inside a layer: only protect heads with highest sensitivity.
- Sensitivity proxy per head: Δattention KL from fakeQuant on that head only OR entropy/margin.
- Compare: layer-level protection vs head-level protection at same byte budget.

A2) INT6 middle tier (if feasible)
- If implementing true INT6 is too hard, approximate with:
  - INT8 but with fewer protected channels (pseudo-INT6),
  - or “grouped INT8” with reduced scale precision.
- Goal: see if a middle precision reduces k*.

A3) Selective value-only INT8
- Keep K at INT4, upgrade V to INT8 for top-k layers.
- Rationale: V quant error may dominate residual distortion.
- Evaluate across k to see if k* drops.

A4) KVSplice only on far tokens (hybrid tier)
- Keep recent window as in S2.
- For far context only: replace INT4 KV with a higher-level memory:
  - either rope_complex for K_far,
  - or KVSplice trained/loaded if available,
  - or a simple segment-averaged V memory.
- Goal: reduce far-context error accumulation without touching all layers.

For each A-method:
- Run k-sweep (or budget sweep) and measure whether minimal k* decreases.
- Log kv_bytes_ratio and PPL.

========================================================
PHASE 3 — Path B: Reduce per-layer damage (3–6h)
========================================================
B1) Better scaling factors
- Compare per-tensor vs per-channel vs per-group scaling for KV quant.
- Evaluate: symmetric vs asymmetric scales for V.
- Choose what minimizes ΔPPL at fixed kv_bytes_ratio.

B2) Per-channel KV quantization
- Implement per-channel (or per-head-dim group) scales for K and V.
- This should reduce sigma4 effectively, lowering k*.
- Benchmark overhead and memory.

B3) Learned calibration (lightweight)
- Freeze base model.
- Learn only quantization scales (and optionally zero points) per layer/head on calibration data.
- Objective: minimize Δlogit KL or ΔNLL with quantized KV cache.
- Keep training short (<= 5k steps).

Success criterion for Path B:
- Show lower k* (min INT8 layers) at tol=3%, OR same k with improved kv_bytes_ratio.

========================================================
PHASE 4 — Path C: Structural modifications (3–6h)
========================================================
C1) rope_complex + mixed precision stacking
- Use rope_complex for K (rank 24 or best stable), and INT4/INT8 schedule for V.
- Goal: reduce damage in K while keeping low bytes.

C2) Attention magnitude normalization
- Add a normalization of attention logits or outputs to reduce amplification of quant noise:
  - clamp logits to a max magnitude,
  - or normalize QK scale per layer.
- Must be inference-only and reversible (no retraining required).

C3) KV rescaling tricks
- Post-quant rescale K/V to match clean statistics:
  - match mean/variance per head,
  - or match norm of V.
- Evaluate if sigma4 drops (measured by Δresidual norms).

Success for Path C:
- reduce k* or improve stability at k below baseline.

========================================================
PHASE 5 — Unified Comparison + Recommendation
========================================================
Create a unified scoreboard:
For each method variant:
- minimal k* for tol=3% (and tol=1%)
- kv_bytes_ratio at k*
- whether it reduces k* vs S2 baseline
- runtime overhead

Declare a winner:
- Primary win: reduces k* by >=2 layers OR achieves kv_bytes_ratio <= 0.30 with PASS.
- Secondary: reduces per-layer sigma4 (as measured) even if k* unchanged.

  - Measured per-layer sigma_int4 and sigma_int8 via
    ||Delta_residual|| under per-layer quantization.

  - INT4/INT8 noise ratio: 12.1x.

Phase 2 (commit 6a2cf11): Path A interventions.
  - A3 V-only INT8: DEAD END. K noise dominates; upgrading
    V to INT8 does nothing (+20-240% at all k).
  - A2 tight-group INT4 g=4: BEST INTERVENTION. k=4
    PASS@3% (max +1.70%). k=0 nearly passes (+4.5% vs
    +429% with g=32). 8x per-layer noise reduction.

Phase 3 (commit 9eda837): Path B per-channel group sweep.
  - g=8: k=4 PASS (max +2.38%), similar to g=32.
  - g=16: paradoxically WORSE than g=32. Non-monotonic
    interaction with head_dim=64.
  - g=32: k=4 PASS (max +1.96%), confirms Phase 0.

Phase 4 (commit 204069e): Path C structural mods.
  - C3 rescaling: DEAD END. Post-quant rescaling of
    mean/std destroys position structure (+7-710%).
  - C2 norm clamp p=0.99: marginal. k=4 improves from
    +1.96% to +1.85%. Does not reduce k*.
  - Fixed bfloat16 dtype bugs in both backends.

Phase 5: Unified comparison.
  - No intervention reduces k* below 4.
  - k-floor is architectural (model sensitivity).
  - A2 tight group g=4 is the only worthwhile intervention
    (reduces per-layer noise, no k* improvement).
