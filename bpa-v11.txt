3.2 Far budget controller must actually spike, not wobble 2→3 forever:
- Instrument k_far(t) and B_far_raw(t).
- Add/modify a stress test that requires retrieving >W_max-away with distractors.
- Define hard/easy spans deterministically and log region-wise metrics:
  span_ppl, span_kv_proxy, span_W_mean, span_k_far_mean, span_latency
- Acceptance for “adaptive far budget works”:
  k_far_max_obs >= 2x baseline in hard spans (example: baseline 2, spike to >=4)
  and returns near baseline outside hard span.

3.3 Make W adaptation more than “slam to W_max and stay there”
- Add a penalty term / governor:
  - Encourage returning to W_min when pressure drops (stronger decay/hysteresis).
- Log “W duty cycle” (% of tokens at each W value).
- The win is not just spikes; it’s spending less W overall without losing quality.

PHASE 4 (1–2 hours): Bandwidth sensitivity check (make latency wins possible)
========================================================
On 124M, latency won’t move much. On a bigger model, it might.

Do two regimes for the chosen model:
- Regime R1: single-sample decode (batch=1) standard
- Regime R2: bandwidth-stressed decode
  Options:
  - increase batch size modestly (2–8) without OOM
  - or increase layers/heads via chosen model size
  - or run multiple sequences in parallel (micro-batching)
Goal: make KV traffic matter.

Report both regimes. Headline claims require R2 showing “KV down and latency not worse” at matched quality.

PHASE 5 (1 hour): bpa_v11_branch_tree.md (think ahead, don’t improvise later)
========================================================
Write a branch tree that maps what the results imply and what v12 should do.

Template (fill with actual observed outcomes):
BRANCH 1: Matched-quality PASS for all L up to 4096 AND latency improves in R2
- Next: push “sublinear scaling” claims carefully:
  - measure effective kept tokens vs L
  - fit exponent only within in-range regime
  - show mechanism: budget spikes aligned with entropy spikes
- Engineering: optimize gate update frequency and scoring kernels.

BRANCH 2: Matched-quality PASS but latency flat (KV down, time same)
- Next: prove it’s compute-bound:
  - profile attention kernel time vs gate time vs other ops
  - test larger model or higher batch until bandwidth-bound
  - consider fused KV gather/scatter or flash-attn path improvements on ROCm.

BRANCH 3: Matched-quality FAIL at higher L (e.g., 4096 fails)
- Next: diagnose what breaks:
  - Is far selection missing crucial tokens? (increase retrieval pool, improve score)
  - Is W too small? (allow adaptive W_l by layer)
  - Add “two-stage”: cheap retrieve then refine retrieve on hard spans only.

BRANCH 4: Far budget does not spike even on adversarial stress
- Next: your pressure signal is not state-aligned:
  - add a second pressure feature (entropy + disagreement)
  - change hard span generator to ensure true far dependency
  - add supervision: label hard tokens and train a tiny controller.

BRANCH 5: Everything works but memory peaks too high / OOM at 8192
- Next: memory tiering experiments:
  - chunked KV, paged KV, offload options, kv quantization for cache only
  - report peak vs L and practical max L on W7900.
