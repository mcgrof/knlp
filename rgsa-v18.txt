```text
rgsa-v18.txt — RGSA v18: Discover State/Usage-Aligned Importance Signals (Head-Level) and Test One Allocation Policy

Context / Ground Truth (v13–v16)

* v13: Learned routing did not reliably beat random; “semantic router” hypothesis falsified at this scale. Uniform L²M-style budget schedules failed in stable regimes. RGSA behaved like a training regularizer: it can stabilize unstable training but hurts stable training when budget is removed uniformly.
* v14–v16: Adam exp_avg_sq (diag Fisher proxy) used as per-layer sensitivity prior for RGSA budget allocation (top_b per layer) was worse, and sign flipped across seeds. Confounds removed (exact budget match, deterministic settings). Conclusion: parameter-space variance measures update geometry, not runtime state value. Close this direction.

v18 Mission

Find a state-aligned / usage-aligned importance signal for RGSA that:
1) correlates with output sensitivity to far-context restriction
2) can be measured cheaply enough to be practical
3) yields an allocation policy that beats uniform allocation at fixed total budget and is stable across seeds

We will NOT “tune forever.” We will run minimal discovery, pick the best signal, and do one policy test.

Primary Goals

A) Instrument head-level usage metrics and head-level impact metrics under uniform RGSA.
B) Determine which cheap usage signal best predicts head importance as measured by an impact metric.
C) Build ONE allocation policy using the best signal and compare to uniform, with strict budget matching, deterministic settings, and multi-seed evaluation.

Non-Goals

* No more Adam/Fisher/exp_avg_sq allocation.
* No router training work; treat routing as a mechanism for constrained selection, not semantics.
* No new kernels, no wall-clock speed claims.

Definitions

RGSA intervention: restricts far-context access by selecting groups/chunks and applying sparse attention only to selected history.
We need importance defined in terms of runtime state: “does this head/layer need far-context capacity to preserve output distribution?”

We will operate at head-level (l,h), not layer-level, because allocation is likely heterogeneous within a layer.

Global Controls (MANDATORY)

* No AUTO mode.
* Pin: batch_size, grad_accum, lr schedule, context length L=1024, dataset shard list, dataloader seed, eval seed.
* Pin: tokens_seen target per run (not time).
* Use identical eval batches across compared variants.
* Strict budget matching:
  sum(top_b_{l,h}) == TOP_B_TOTAL exactly (same total tokens/query across policies).
* Logging:
  tokens_seen, val_ppl, tokens/query stats, attention FLOPs proxy, and all new head-level metrics below.

Key Metrics (Head-Level)

USAGE METRICS (cheap, online)
For each head (l,h), computed during eval on a small batch:
1) Far-attention mass:
   far_mass[l,h] = attention_weight_sum to positions outside local_window / total attention mass
2) Attention entropy over attended positions:
   attn_entropy[l,h] (normalized)
3) Attention concentration:
   max_weight[l,h], top1_mass[l,h], or Gini coefficient
4) Retrieval set stability (if applicable with groups):
   run the same eval batch twice with the same seed and once with a tiny perturbation (e.g., different pack boundaries or slight input reorder) and measure Jaccard overlap of selected groups per head: jaccard[l,h]

IMPACT METRICS (causal-ish, medium cost)
We will measure importance via a “drop-impact KL” proxy:
- Compute logits for uniform RGSA: logits_uniform
- For a subset of heads, create a “drop head far-context” variant where head (l,h) is forced to attend only within local_window (or set top_b_{l,h}=0) while other heads remain uniform.
- Compute logits_drop(l,h)
Define:
   impact_kl[l,h] = KL(softmax(logits_uniform) || softmax(logits_drop(l,h))) averaged over tokens

To control cost:
- Compute impact_kl only for a subset of heads per eval step (round-robin).
- Maintain an EMA table impact_kl_ema[l,h].

Optional: marginal utility signal (even better, if feasible):
- Add +1 unit budget to head (l,h) (or +1 chunk) and measure KL decrease vs uniform.
This is “benefit per budget,” but more complex. Only implement if drop-impact works and is stable.

Phase Plan

PHASE 0 — Plumbing Sanity (FAST, 1 seed, short)
Goal: verify logging and strict budget matching.

Tasks:
0.1 Add head-level metric logging hooks (far_mass, entropy, concentration).
0.2 Implement exact head-level budget matcher:
    Given weights w[l,h], produce integer top_b_{l,h} with exact sum TOP_B_TOTAL.
0.3 Verify uniform policy yields a uniform allocation tensor (all heads equal or per-layer equal per existing design).

Run:
- uniform RGSA for ~20–30 minutes or fixed tokens_seen small
Acceptance:
- metrics appear in W&B and are non-degenerate (not all zeros)
- allocation sums exactly match TOP_B_TOTAL
- compile still enabled, no graph breaks

PHASE 1 — Measure Head Importance via Drop-Impact KL (DISCOVERY)
Goal: obtain a stable impact_kl_ema[l,h] table.

Tasks:
1.1 Implement “drop head far-context” for a head (l,h):
    - Force head to only local_window (top_b_{l,h}=0 or mask far groups)
    - Keep total budget fixed by redistributing that head’s budget to others ONLY in measurement mode OR keep total budget reduced but record it explicitly.
Preferred: keep total budget constant by reassigning dropped head budget uniformly across remaining heads for measurement runs.
1.2 Implement round-robin schedule:
    - Each eval step: pick K heads (e.g., K=8) to measure impact_kl
    - Update impact_kl_ema
1.3 Add W&B plots:
    - heatmap of impact_kl_ema[l,h]
    - top-10 heads by impact_kl_ema

Runs:
- 1 seed, small tokens_seen (enough eval steps to cover most heads once or twice)
Acceptance:
- impact_kl_ema shows structure (not pure noise)
- top heads are consistent across adjacent evals (stability check)

PHASE 2 — Find Predictors: Which Usage Signals Track Impact?
Goal: decide what to use for allocation policy.

Tasks:
2.1 For heads with measured impact_kl_ema, compute correlation / rank agreement between:
    impact_kl_ema and far_mass
    impact_kl_ema and attn_entropy
    impact_kl_ema and concentration
    impact_kl_ema and jaccard stability (if available)
Use:
- Spearman rank correlation
- Top-K overlap (e.g., overlap of top-10 heads)
2.2 Choose ONE predictor signal P[l,h] to drive policy:
Priority order:
- If far_mass strongly predicts impact_kl -> use far_mass
- Else if concentration/entropy predicts -> use that
- Else if retrieval stability predicts -> use that
- Else fall back to impact_kl_ema itself as the oracle (but note it is more expensive)

Deliverable:
- Short table in the report: correlations and chosen signal

PHASE 3 — Policy Test: Uniform vs Signal-Weighted Allocation (MULTI-SEED)
Goal: test whether state/usage-aligned allocation beats uniform at fixed total budget.

Define policy:
- weights w[l,h] = (P[l,h] + eps)^gamma
- gamma in {0.5, 1.0} (start with 1.0; 0.5 if extreme skew)
- exact-match to TOP_B_TOTAL using matcher

Variants:
A) uniform allocation
B) P-weighted allocation (chosen P)
Optional:
C) inverted P (sanity: if P is causal, inverted should worsen)

Runs:
- seeds 1,2,3
- tokens_seen matched (same stop condition)
- identical eval schedule and eval set

Metrics:
- mean±std val_ppl at matched tokens_seen
- tail risk: worst-seed PPL
- if feasible, KL(uniform vs policy) or KL(policy vs uniform) on eval
- tokens/query distribution (must be matched)

Acceptance:
- Policy must beat uniform in mean OR reduce tail risk without increasing mean.
- If effect flips by seed (sign reversal), declare predictor unusable at this horizon and stop.

Stopping Rules

* If Phase 1 impact_kl is pure noise (no stable structure), stop and increase eval batch size slightly (only once). If still noise, stop the project.
* If Phase 2 shows no predictor correlates with impact_kl, and impact_kl itself is too expensive, stop and write: “No usable cheap signal found.”
* If Phase 3 shows no improvement or sign flips by seed, stop and conclude: “Uniform allocation is near-optimal under current RGSA design and regime.”
```
