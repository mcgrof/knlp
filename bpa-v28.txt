PHASE 2 — INT4 attention path: remove the Python tax (8–16h)
========================================================
Goal:
Replace the current Python-level dequant/attention overhead with a compiled path that is worthy of a systems paper.

Important constraint:
Do not try to write a heroic production kernel from bare metal if an existing compiled path can be adapted. Prefer the shortest path to a credible compiled implementation.

Acceptable solutions in order of preference:
A) integrate/adapt an existing CUDA/Triton compiled INT4 KV attention path
B) build a Triton kernel that fuses INT4 unpack + scale apply + attention-relevant load/dequant on GPU
C) if true fusion is too much, build a compiled near-fused path that removes Python loops and demonstrates the overhead reduction clearly

Unacceptable outcome:
- another Python wrapper with a different name

Tasks:
2.1) Benchmark the current path first:
- measure per-token latency and kernel breakdown for:
  - INT8 baseline
  - current g32 mixed precision path
  - current amortized-scale path
- identify where the overhead lives

2.2) Implement the compiled INT4 path for at least one headline backend:
- standard g32 mixed precision is the minimum target
- amortized g8 S8 is a stretch target if feasible

2.3) Validate correctness:
- compare numerical output against current reference implementation on a fixed small batch
- ensure quality metrics do not regress materially relative to the reference path

2.4) Benchmark on rqv under the canonical protocol:
- same model(s)
- same L
- same seeds

Outputs:
- docs/bpa_v28_kernel_report.md
- artifacts/v28/kernel_benchmarks.json

Success criteria:
Minimum acceptable:
- compiled path reduces the current Python overhead clearly
- compiled INT4 path is no slower than the current Python path
Strong success:
- compiled path gets within ~2% of INT8 baseline latency or better in at least one tested regime
Stretch:
- compiled path exposes an actual latency gain in a bandwidth-stressed regime

If compiled path cannot be completed:
- document blocker precisely
- do not bluff
