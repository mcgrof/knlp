rgsa-v14.txt — FineWebEdu RGSA v14: Variance-Weighted L²M + Sensitivity-Aware Budgeting (A2/A3) with KL-Drift Monitoring

Context / Ground Truth So Far (v13 Recap)

* Phase 1 (FineWebEdu): RGSA variants beat an unstable baseline; learned routing did not reliably beat random; static constraints were most stable.
* Phase 2 (Budget schedules): Uniform budgeted RGSA variants performed WORSE than a stable baseline; seed variance exploded; no compute–quality frontier improvement.
* Interpretation: “Uniform capacity reduction fails once training is stable.” The failure mode is likely misallocation (budget removed from critical subspaces), not just insufficient total budget.
* New insight / goal: Use Adam variance (v̂, diagonal Fisher proxy per Squisher) as a static sensitivity prior to allocate attention capacity where it matters, and test whether this fixes the v13 failure mode.

Primary Goals for v14

1. Implement variance summaries S (per-layer, optionally per-head) derived from Adam exp_avg_sq (v̂) at checkpoints, and log them.
2. Add two new ablations:
   - A2: Variance-only allocation (fixed total budget, skew allocation by S)
   - A3: Variance-weighted + L²M-style total budget schedule (skew allocation by S, scale total budget by L or simulated L)
3. Determine whether variance-weighted allocation reduces KL drift / PPL degradation relative to uniform budgets, especially in stable-baseline regimes where v13 failed.
4. Measure whether variance-weighting reduces tail risk (catastrophic seeds), not just mean PPL.
5. Produce a clean report that reuses v13 as A0/A1 evidence and adds A2/A3 as the missing tests.

Non-Goals

* No new semantic routing training work (v13 falsified “router finds semantics” at this scale).
* No custom CUDA kernels.
* No new dataset research beyond FineWebEdu staging already implemented.

Definitions / Hypothesis (New Framing)

RGSA = Route-Gated Sparse Attention: group history, router scores groups, gate selects subset, sparse attention applied on subset.

We test the two-level constraint:

Macro (L²M): total effective routed history budget B(L) must grow with L at least sublinearly (power law).

Micro (Variance prior): budget should be allocated non-uniformly across layers/heads using sensitivity S derived from Adam v̂.

S is NOT “KV Fisher.” S is a static training-derived prior for which attention pathways are expensive to compress.

Core v14 prediction at fixed memory:
A3 (variance + macro scaling) should beat A1 (macro-only uniform) and A0 (fixed uniform), and should have lower tail risk than random routing.

Ablation Taxonomy (Map to v13 + New v14)

A0: Fixed RGSA (uniform budgets, fixed chunk/top_b/local)  -> already covered by v13 static configs
A1: Macro-only RGSA (uniform budgets with schedules)        -> v13 Phase 2 (failure case)
A2: Variance-only RGSA (fixed total budget, variance skew)  -> NEW in v14
A3: Variance + Macro RGSA (total schedule + variance skew)  -> NEW in v14

Expected ordering for KL stability at fixed memory:
A3 > A2 > A1 > A0  (if variance signal is real; otherwise A2/A3 collapse to A1)

Important: We keep learned vs random routing as a side diagnostic ONLY if needed; primary focus is allocation, not semantics.

NEW: Variance (Sensitivity) Extraction and Storage

Goal: compute static sensitivity summaries S from Adam optimizer exp_avg_sq (v̂) at checkpoint time.

Implementation requirements:
1) Add a utility to load optimizer state and extract exp_avg_sq for relevant parameters.
2) Aggregate into:
   - S_layer[l] = sum(exp_avg_sq) over attention projection params in layer l (Wq, Wk, Wv, Wo) and any RGSA router/gating params in that layer.
   - Optionally S_head[l,h] if and only if head reshaping is unambiguous; otherwise stay per-layer first.
3) Save to a tiny JSON alongside each checkpoint:
   - sensitivity.json with fields: step, tokens_seen, S_layer[], (optional S_head[][]), summary stats (p50/p90/max), and the mapping assumptions used.
4) Log S_layer (and optionally S_head) into W&B at checkpoint time:
   - heatmap plot per run
   - top-k most sensitive layers list

If optimizer state is not saved for some runs, add a “save_optimizer_state” flag for the runs that matter.

NEW: Variance-Weighted Budget Allocation Policy

We need a way to operationalize “budget per layer” inside RGSA.

Define:
* Total budget in tokens/query: B_total = local_window + top_b * chunk_size
* Per-layer budget multiplier: w_l = S_l^α / sum(S^α)

Then implement one practical allocation mechanism (keep minimal code changes):
Option 1 (preferred): Per-layer top_b scaling
* top_b_l = clamp(round(top_b_base * w_l * L_mult), min=top_b_min, max=top_b_max)
* Keep chunk_size fixed per run for simplicity.

Option 2: Per-layer chunk_size scaling
* chunk_l = choose from {32,64,128} by w_l quantiles; keep top_b fixed.

Option 3: Per-layer “effective gating cap”
* Same router scores, but cap number of selected chunks per layer by top_b_l.

Start with Option 1 (per-layer top_b) because it’s the least invasive.

Hyperparameters for variance policy:
* α_var in {0.0, 0.5, 1.0} (0.0 is the uniform control)
* top_b_min = 1 (or 2)
* top_b_max = 16 (or existing safe cap)
* Normalize w_l using median(S) if needed for numeric stability.

Important: Keep total expected tokens/query approximately fixed when comparing methods.
If per-layer allocation changes totals, renormalize by scaling top_b_base so sum(top_b_l) matches the uniform baseline.

NEW: KL-Drift Monitoring (Optional but Strongly Recommended)

We want a distributional drift proxy during training/eval without huge overhead.

Add one of these:
1) Logit KL on eval mini-batch:
   - compute KL between baseline (full attention) logits and RGSA logits on the same eval inputs.
   - Implementation: run baseline forward pass at eval time for a small eval subset (e.g., 1/10th eval iters) to control overhead.
   - Metric: mean KL per token, plus p95 KL.

If baseline forward pass is too expensive:
2) Proxy drift metrics:
   - cosine similarity of hidden states at final layer between baseline and RGSA
   - attention mass distribution shift stats

Preferred is (1) if feasible. It directly supports the “KL stability” claim.

W&B logging additions:
* val_logit_kl_mean, val_logit_kl_p95
* sensitivity_alpha (α_var), per-layer budget histogram
* tail metrics: p95/p99 val PPL across seeds once aggregated
