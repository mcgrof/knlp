Context / Ground Truth (v13–v14)

* v13: Learned routing ≈ random routing (no reliable semantic routing). Uniform L²M-style budget schedules failed in stable regimes. RGSA behaved like a training regularizer: it helped unstable training, hurt stable training.

A.1 Implement Exact Budget Matching (No Rounding Drift)

Problem:
- prior allocations differed by ~2% budget due to rounding (94 vs 96 total).
- any mismatch weakens claims.

Requirement:
- implement a renormalization step that guarantees:
  sum(top_b_l) == TOP_B_TOTAL exactly (for all allocations)

Suggested method:
1) Compute unnormalized weights w_l (or w_{l,h})
2) Compute raw allocations a_l = w_l * TOP_B_TOTAL
3) Take floor: top_b_l = floor(a_l)
4) Distribute remaining tokens (TOP_B_TOTAL - sum(top_b_l)) to layers with largest fractional remainders
5) Apply caps (min/max), then re-balance again to preserve exact sum

PART B — Next Frontier: State-Based Importance (Drop-Impact KL)
-----------------------------------------

New Objective (based on v13–v14 findings)

We hypothesize:
- v13/v14 failures are primarily due to misallocation of budget across model pathways.
- Parameter-space update magnitude (exp_avg_sq) is not the right importance proxy.
- A correct importance proxy must be defined in terms of RGSA’s intervention: restricting access to past context.

Therefore we will define a causal, state-aligned signal:

DROP-IMPACT IMPORTANCE:
“How much does output distribution shift if we remove far-context access for a given layer/head?”

We use this to allocate RGSA budget.

B.1 Define Baseline and RGSA Variants for Measurement

We need a stable reference and a stable RGSA config.

Reference:
- full attention baseline model forward at eval time (teacher logits)

RGSA candidate:
- uniform RGSA (alpha=0.0) with fixed chunk/local/top_b_total, stable policy

We will NOT train new routers; this is evaluation-time measurement + budget allocation.

B.2 Implement Drop-Impact KL Measurement (Cheap Oracle Label)

At evaluation on a small batch:

For each layer l (start per-layer; head-level optional later):

1) Compute teacher logits with full attention: logits_full
2) Compute RGSA logits under uniform allocation: logits_rgsa_uniform
3) Compute DROP variant:
   - disable far-context retrieval for layer l only (or set top_b_l=0 for that layer, keep local_window)
   - keep all other layers same as uniform RGSA
   - compute logits_drop_l

Then compute:
- KL_full_vs_uniform = KL(softmax(logits_full) || softmax(logits_rgsa_uniform))
- KL_uniform_vs_drop_l = KL(softmax(logits_rgsa_uniform) || softmax(logits_drop_l))  (or use full as reference consistently)

Define importance score:
- I_l = mean KL increase when dropping layer l’s far context
Optionally also track p95 I_l.

Important: Keep overhead bounded:
- measure I_l only on a small eval subset (e.g., 32 sequences, 1 eval step every N evals)
- optionally compute I_l for a subset of layers per step (round-robin) and accumulate EMA

Log to W&B:
- I_l heatmap
- top-k layers by I_l

B.3 Convert Importance into Budget Allocation

Given fixed TOP_B_TOTAL, allocate per layer:

w_l = I_l^γ / sum(I^γ)
top_b_l = exact-budget-matching(w_l, TOP_B_TOTAL)

Hyperparameter:
- γ in {0.5, 1.0} (start with 1.0)
- Always compare to uniform.

This is the new “importance-weighted allocation,” but now importance is defined causally in state terms.

B.4 Experiments (bounded)

Phase B1: Measure-only validation (no allocation)
- Run uniform RGSA and compute I_l
Acceptance:
- I_l should not be degenerate (not all equal, not random noise)
- I_l should be at least moderately stable across nearby steps

Phase B2: Allocation test (uniform vs impact-weighted)
Runs (seeds 1,2,3; tokens_seen matched):
1) uniform RGSA allocation
2) impact-weighted allocation (I_l, γ=1.0)
3) optional: smoothed impact allocation (EMA of I_l)

Metrics:
- val PPL mean/std
- tail risk (worst-seed, p95)
- KL drift vs teacher (if teacher logits are computed anyway, log KL(full || rgsa))
- tokens/query distribution (must match)

Acceptance:
- if impact-weighted beats uniform (mean or tails) at fixed budget, misallocation is fixable and we proceed.
- if it does not, conclude uniform is near-optimal for this RGSA family at this scale.

B.5 Optional Extension (only if per-layer works)

If per-layer impact-weight helps:
- extend to per-head importance I_{l,h} by dropping head h’s far-context access within a layer.
- allocate top_b_{l,h} similarly, with strict budget matching.
- keep it bounded: only 2–3 layers or only top-k heads first.

### Task 3: Part A Confirmation Runs - CONTRADICTION FOUND!

Run with exact budget matching (all sum to 96), 200 iters, seed 1:

| Allocation | top_b_per_layer | PPL | vs Uniform |
|------------|-----------------|-----|------------|
| uniform | [8]*12 | 701.82 | baseline |
| variance_0.5 | [16,16,16,12,5,5,5,5,4,4,4,4] | 733.92 | +4.6% (worse) |
| **inverted_0.5** | [2,4,7,8,8,9,9,9,10,10,10,10] | **553.10** | **-21.2%** (better!) |

### CRITICAL FINDING

**The v14 "negative result" was WRONG due to budget mismatch!**

With exact budget matching:
- Variance weighting (more to early layers) is still slightly worse (+4.6%)
- BUT inverted weighting (more to later layers) is **21% better**!

This means:
1. The sensitivity signal IS useful, but INVERTED
2. Layer 0 being highly sensitive means it needs LESS routed attention, not more
3. Later layers (lower sensitivity) benefit from more far-context routing

### Interpretation

The sensitivity (Adam exp_avg_sq) measures parameter update magnitude.
High sensitivity at layer 0 may indicate:
- Early layers are learning low-level features rapidly
- They don't need far-context (works fine with local attention)
- Later layers need far-context for semantic reasoning

### SEED 2 RESULTS - CONTRADICTION REVERSED!

| Allocation | Seed 1 PPL | Seed 2 PPL | Difference |
|------------|------------|------------|------------|
| uniform | 701.82 | 664.76 | 5.5% |
| inverted_0.5 | 553.10 | 843.27 | **52%** |

Seed 1 showed inverted 21% better, but Seed 2 shows inverted 27% WORSE!

### FINAL CONCLUSION

**Seed variance completely dominates allocation effects at 200 iters.**

The v14 negative result STANDS:
- Variance weighting doesn't reliably help
- Inverted weighting doesn't reliably help either
- The 52% swing across seeds means any single-seed result is unreliable
