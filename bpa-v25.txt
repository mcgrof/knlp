PHASE 1 — Write the narrative (core deliverable) (4–8h)
========================================================
Write docs/bpa_v25_technical_narrative.md with this structure and content rules.

1) Executive Summary (half page max)
- What problem: long context KV cost explodes; want mechanisms that reduce effective KV usage / bytes while preserving quality.
- What we found: structural floors exist; accumulation dominates; only O(1) “sink layers” need protection; kv_ratio trends to ~0.28–0.30 with INT4+small k; W7900 is compute-bound so benefits are capacity not latency; H100 is needed for bandwidth-bound latency proof.
- Must include the v24 O(1) headline: k*(3%)=2 for D=24 and D=28 with k/D decreasing. :contentReference[oaicite:5]{index=5}

2) Problem Statement and Original Hypothesis (L²M-motivated)
- State the L²M / mutual-information motivation plainly:
  “Predictive information grows sublinearly in L, so far-context needs should be conditional, not proportional to L.”
- Define what we intended to measure: effective KV usage vs L; exponents after subtracting geometric floors.
- Clarify what we mean by “sublinear”: exponent beta < 1 on effective KV bytes after floor removal.
- Explicitly state the initial bias: we expected gating/budget adaptation alone could deliver sublinear scaling curves.

3) Definitions and Metrics (no ambiguity)
Define and stick to these:
- L: context length, and which values used (e.g., 8K/16K/32K).
- D: number of transformer layers.
- kv_ratio: KV bytes per token vs dense baseline, including metadata (scales/zeros).
- ε: relative PPL tolerance, PASS_ε meaning (PPL_q - PPL_dense)/PPL_dense <= ε.
- k: number of protected layers at higher precision (INT8), chosen by ranking.
- k*(ε): minimum k that passes PASS_ε across all seeds and L set.
This section must exactly match v24 usage. :contentReference[oaicite:6]{index=6}

4) Timeline of Empirical Discoveries (the “bias evolution” section)
This is mandatory and must be blunt:
- Early: “KV kept” metrics and prettier beta plots.
- v8/v9: discovered local window (W) dominates and creates a geometric floor; chasing beta without subtracting floor is misleading.
- Mid: discovered gate overhead and decode-time regimes matter (compute-bound vs bandwidth-bound).
- Later: discovered quantization noise accumulation across depth creates a k-floor and dominates failure modes; V-only protection doesn’t work because K/RoPE dominates.
- v21: discovered honest byte accounting and scale metadata overhead makes small quant groups expensive.
- v22/v23: amortization reduces metadata overhead; throughput/capacity benefits observed; still compute-bound on W7900.
- v24: formal accumulation model + k(D,ε) bound + O(1) evidence.
For each bullet, cite a specific version report file and include one concrete numeric example where possible.

5) The Accumulation Theory (formal section)
This section must be written as a mini research note:
- Present residual recursion and perturbation.
- Present linearization leading to: ||e_D|| ≤ Σ α_ℓ σ_ℓ.
- Explain meaning of α_ℓ (amplification/importance) and σ_ℓ (noise magnitude).
- Explain why greedy top-k protection is optimal under this bound.
- Explain O(1) condition: if α mass is concentrated in O(1) sink layers, then k* stays constant as D grows.
Use v24 text as source of truth; paraphrase, do not invent. :contentReference[oaicite:7]{index=7}

6) Empirical Evidence for O(1) (data-first)
Include a table like v24’s:
- qwen05b (D=24): k*(3%)=2, k*(1%)=None
- qwen15b (D=28): k*(3%)=2, k*(1%)=3
and explicit statement: k* unchanged while D increased → supports O(1). :contentReference[oaicite:8]{index=8} :contentReference[oaicite:9]{index=9}

Also include the sink-layer evidence:
- oracle sensitivity shows layer 0 dominates (0.5B +23%, 1.5B +824%).
- tail fractions drop sharply with model scale (cite exact values from v24 report).
Explain in one paragraph why sink layers plausibly arise in RoPE transformers (attention sink role), without overclaiming causality. :contentReference[oaicite:10]{index=10}

7) What We Have Now (current mechanism, stated plainly)
Describe the current “winning recipe” in generic terms:
- Use INT4 KV for most layers.
- Protect top-k layers (k small, ~2–4) at INT8, chosen by oracle sensitivity ranking.
- This achieves kv_ratio ~0.28–0.30 at ε=3% for tested models.
- On W7900 (compute-bound), primary benefit is capacity/throughput, not latency.
Do not claim latency wins on W7900 unless you have a benchmark table proving it.

8) The Gap to “information-theoretic lower bounds”
Be honest and specific:
- Our bound is an operational bound on error accumulation, not MI lower bound.
- L²M informs expectation of conditional far-context needs, but the observed bottleneck is quantization noise accumulation and sink-layer sensitivity.
- The gap now is: can we align conditional retrieval/gating with the O(1) sink-layer compression story to push beyond byte floors (~0.28)?
List what remains unknown.

9) Limitations and What Would Falsify the Story
This section is mandatory:
- O(1) is only supported on D=24 and D=28. Needs validation at larger D (7B/8B).
- ε=1% can require higher k (seen at 1.5B).
- Proxy rankings can fail (residual proxy explosion); oracle is expensive.
- Scale metadata overhead can erase gains for small group sizes unless amortized.
Include falsification criteria: if k* grows proportionally with D on 7B/8B, O(1) fails.

10) H100 Test Plan (separate, concrete section)
This is the final section and must be “runbook-like”:
- Objectives:
  (i) validate O(1) on 7B/8B (D~32+) at L=32K–64K
  (ii) confirm kv_ratio improves toward ~0.27–0.28
  (iii) demonstrate bandwidth-bound latency reduction in at least one regime
- Experiment matrix:
  Models: Llama-3-8B or equivalent long-context model; plus one 7B baseline if needed.
  L: 16K, 32K, 64K (if supported)
  batch: 1, 8, 16, 32 (aim for bandwidth bound)
  k: {0,1,2,3,4,6,8}
  method: INT8-all baseline, INT4 g=32 + top-k protection, (optional) amortization if it’s stable and implemented.
- Ranking method:
  Start with cheap 1-seed oracle sweep to rank layers, then 3-seed confirm on top candidates.
- Acceptance criteria (must be numeric):
  A) O(1): k*(3%) <= 4 at D>=32 and does not scale with D over tested models.
  B) kv_ratio: <=0.28–0.30 at ε=3%.
  C) latency: at least one bandwidth-bound regime shows >=10% ms/token reduction vs INT8 baseline.
  D) throughput: >=20% more concurrent sequences at fixed latency OR fixed memory.
Tie these directly to the v24 branch tree rationale. :contentReference[oaicite:11]{index=11}
