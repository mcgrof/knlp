bpa-v15.txt — “Structural Bottleneck Attack: RoPE-aware, K/V-separable, logit-space memory”
Mission: v14b proved the distortion wall: INT8 works (precision compression), low-rank and KVSplice (structure compression) fail. v15 must attack the TRUE bottleneck: positional phase + value manifold + layer sensitivity. This is not “tune thresholds.” This is “change the compression representation so it can exist without collapse.”

Non-negotiables:
- GPU: AMD W7900 ROCm. Headline runs GPU-only.
- Model: use the SAME “real model of value” you used for v14b (your custom GPT-2 / knlp model). If Qwen was used previously, replicate on Qwen too, but HEADLINE is your real model.
- Contexts: L ∈ {8192, 16384, 32768} (plus one smaller sanity point L=4096). Decode_steps >= 256. Seeds 0/1/2.
- Regimes: r1 batch=1, r2 microbatch=4 (auto-downshift allowed but logged).
- Matched-quality PASS at tol=3% (and report tol=1% if any method passes).

Core hypothesis (structural):
1) K is constrained by RoPE phase/geometry; naive pooling/low-rank destroys phase info.
2) V is full-dimensional semantic mixture; low-rank is wrong primitive.
3) Any “memory” must preserve attention logits structure (QK^T) or preserve a surrogate (logit-space) rather than approximating KV directly.
4) Compression must be layer-aware (late layers are more sensitive).

Unified harness:
- eval_harness.py supports kv_backends: dense, int8, int4 (for sanity), rope_aware_lowrankK + int8V, logit_space_memory, per_layer_kvsplice, head_clustered_kvsplice, phase_preserving_kvsplice
- Every run logs JSON with: ppl, ppl_delta%, latency p50/p95, tok/s, kv_bytes_proxy/token, peak mem, overhead breakdown, tier fractions, catastrophic flags.
- For every method, compute a rate–distortion curve by sweeping its key knob.

Deliverables:
- results/v15/* logs + CSV summaries
- bpa_v15_final_report.md
- bpa_v15_branch_tree.md
- artifacts/ (trained weights for learned modules)
- a “frontier scoreboard” ranking methods by (PASS @ 3%) and then best kv_bytes_ratio + latency_ratio.

PHASE 1 (2–4h) — K/V-SEPARABLE compression: “Don’t low-rank V. Ever.”
========================================================
Method M1: Rope-aware Low-rank for K ONLY + INT8 for V
Why: v14b suggests K has compressible structure; V does not. K must preserve RoPE phase.
Implement:
- For each layer ℓ and head h:
  - Compress K via low-rank in a RoPE-aligned basis (see below), keep V in INT8.
- Sweep rank budgets: r ∈ {d/2, d/4, d/8} per head OR global budget.

RoPE-aligned basis options (implement ALL, pick winner by results):
A) Pre-RoPE de-rotation (if accessible):
   - If model uses RoPE: attempt to “unrotate” K back to a canonical phase before compression, then rerotate on retrieval.
   - This preserves phase. If you can’t access pre-rotation K, approximate by applying inverse RoPE at the token position.
B) Complex-plane grouping:
   - Treat RoPE pairs as complex numbers; compress in complex space with low-rank on magnitude/angle separated components.
C) Frequency-band splitting:
   - Split head dim into low/high frequency bands (RoPE uses frequencies). Compress high-frequency band less aggressively.

Evaluation:
- Must PASS tol=3% at L>=8192 for at least one configuration.
- Target kv_bytes_ratio <= 0.65 vs dense (better than pure INT8 only if possible).
- Must not regress p50 latency in bandwidth-bound regime.

Branch:
- If M1 PASS and improves kv_bytes beyond INT8 (and latency not worse): prioritize M1 for tiering.
- If M1 fails: conclude K compression must preserve logits directly (move to Phase 2/3 emphasis).

========================================================
PHASE 2 (3–5h) — LOGIT-SPACE MEMORY (stop approximating KV; approximate attention logits)
========================================================
Key idea: the model cares about QK^T. So compress a representation that directly approximates logits contribution from far context.

Method M2: Logit-space Nyström / landmark approximation
- Choose m “landmark” tokens among far context (<= t-W_min).
- Approximate attention to far tokens using Nyström:
  - Use K_landmarks and compute approx logits for all far tokens via low-rank in logit space.
- Sweep m ∈ {64,128,256,512} (depending on L).

Method M3: Head-wise far-context “logit sketch”
- Maintain per head a small sketch S that approximates K_far distribution:
  - e.g., random features / kernel sketch (cosine features) for dot-product attention
  - approximate exp(QK^T) contributions
- This is advanced; implement a simple dot-product sketch baseline first.

Training (optional but recommended):
- Freeze base model. Train only sketch parameters to minimize Δattention KL between dense attention distributions and approximated distributions for sampled decode steps.
- If training too heavy, run untrained variants + note results, then train best candidate.

Acceptance:
- Any M2/M3 PASS at tol=3% at L>=8192 while achieving kv_bytes_ratio < INT8 baseline is a big win.
- If it only matches INT8, still valuable if it enables >2× later.

Branch:
- If M2/M3 PASS: proceed to combine with INT8 (Phase 5).
- If M2/M3 FAIL: implies logits too sensitive; must do per-layer learned KVSplice with phase preservation (Phase 3).

========================================================
PHASE 3 (4–6h) — “KVSplice, but not dumb”: per-layer, position-aware, phase-preserving, trained on Δattention KL
========================================================
This is the corrected KVSplice frontier. v14b KVSplice failed because it was global/shared and lost RoPE phase.

Implement these KVSplice variants (ALL trained; no untrained allowed):
M4: Per-layer KVSplice (separate parameters per layer)
- Each layer gets its own segmenter/projection.
- Far tokens compressed into M memory segments.
- Use Δattention KL loss at that layer.

M5: Phase-preserving KVSplice
- Incorporate position encoding explicitly:
  - memory segments store both content and a phase/position summary
  - retrieval applies RoPE-compatible positioning
- Option: keep “phase anchors”: store a small set of exact K for certain positions, splice the rest.

M6: Head-clustered KVSplice
- Cluster heads by behavior (use attention entropy / variance stats).
- Use separate splicers per cluster.

Training protocol:
- Teacher: dense attention logits.
- Student: near full KV + far replaced with spliced memory.
- Loss: Δattention KL (with τ in {1,2}) + optional attention-output MSE.
- Curriculum on M (memory segments):
  - start mild: M = T_far/2
  - then tighter: T_far/4
  - then T_far/8
- Train budget: 20k steps total across curriculum, early stop on validation.
- Mixed precision, NaN guards, checkpoint per stage.

Sweep:
- Memory segments M: {T_far/2, T_far/4, T_far/8} for each L band
- Layer subsets: train all layers vs last 1/3 layers only (to save compute); compare.

Acceptance:
- PASS tol=3% at L>=8192 for at least one M configuration.
- Must beat naive KVSplice failure by huge margin (obviously).
- If PASS at L=32768 with kv_bytes_ratio <= 0.6 and no latency regression: this is a primary win.

Branch:
- If any of M4–M6 PASS at 32K: KVSplice is viable long-term memory (Tier2) with the right structure.
- If only passes at 8K/16K: likely needs more capacity (more M, per-layer specialized, or partially unfreeze model).
- If fails everywhere: conclude KVSplice is mismatched for this architecture without deeper changes; prioritize logit-space methods + quant.

========================================================
PHASE 4 (2–3h) — Layer sensitivity profiling (so compression becomes layer-aware by evidence)
========================================================
Goal: identify which layers tolerate compression error.
Procedure:
- For each layer ℓ, apply INT8/INT4 to KV only in that layer while keeping others dense (or a controlled perturbation).
- Measure ΔPPL and Δlatency.
Output:
- layer_sensitivity.json and a ranked list of layers by sensitivity.
Use this to:
- allocate more bits/rank/M to sensitive layers
- compress aggressive in tolerant layers

Acceptance:
- Produce a clear “sensitivity curve” and use it to define an adaptive schedule for at least one method in Phase 5.

Run final evaluation:

Acceptance:

- Must improve or not regress p50 latency in bandwidth-bound regime
