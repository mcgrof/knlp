bp-v3.txt — BPA v3: Measure KV Cache Memory, FLOPs, and Time Gains (Quality Non-Regression)

ONE-LINE DEFINITION (do not lose this):
Boundary-Pressure Attention (BPA) is a conditional rate controller for the model’s history channel, attempting to match memory usage to the mutual information structure of language rather than to sequence length.

GOAL (primary):
Quantify BPA’s reductions in KV cache waste (memory footprint + bandwidth/traffic proxies) while maintaining quality (PPL non-regression). Any PPL improvement is a bonus, not the objective.

SECONDARY:
Quantify algorithmic savings (tokens/query, FLOPs proxy) and real savings (ms/token, tokens/sec). If algorithmic savings do not translate to wall time due to dense kernels, document that explicitly.

PHASE 0 — Baseline formulas and sanity checks
================================================================================
Implement analytic KV accounting and sanity-check against known transformer shapes.

Add utility functions (prefer in `utils/kv_accounting.py`):
- kv_bytes_per_token(n_layers, d_model, bytes_per_elem):
    returns 2 * n_layers * d_model * bytes_per_elem
- kv_bytes_written_per_token(...):
    same as above (write new K,V per generated token per layer)
- kv_bytes_read_per_token(attended_tokens_per_layer, n_layers, d_model, bytes_per_elem):
    approximate read cost for attention: attended_tokens * d_model * 2 * bytes_per_elem * n_layers
  (Note: reads depend on attended tokens; writes depend on new token)
- effective_kept_tokens_per_query(local_window, far_tokens_selected):
    local_window + far_tokens_selected

Sanity: print (and log) expected KV bytes/token for the current model config at fp16/bf16.

--- Phase 4: Decisions (DONE) ---

L=512 verdict: GO.
BPA learned gate reduces KV read traffic by 27.4% with only
0.9% PPL regression. The gate is selective and quality is
preserved. This result justifies proceeding to longer sequences.

L=1024 verdict: CONDITIONAL.
KV read savings are 53.3% but PPL regresses 11.3%. Far-context
genuinely helps at L=1024 and the gate is not selective enough
to preserve quality while skipping it. V2 (learned, PPL=272.4)
barely beats V1 (local-only, PPL=275.9), meaning the gate adds
minimal value — it enables far-context ~29% of the time but the
quality loss from the remaining 71% that stays local-only
dominates. V2 does beat V3 (random, PPL=281.0), confirming the
gate is better than random, but not good enough.

Wall time: NO savings materialized.
Dense attention kernels do not skip masked-out positions. V0,
V1, and V3 all run at ~0.26-0.38 ms/tok. V2 is 2-3x slower
(0.64-0.89 ms/tok) due to gate feature extraction overhead.
True time savings require sparse attention kernels (FlashAttention
with block-sparse masks or similar).

Peak KV allocation: UNCHANGED.
The dense cache layout allocates the full L*d buffer regardless
of gating. BPA only reduces effective read traffic and compute,
not the allocation. True memory savings require a sparse or
compacted cache layout.

Next steps (if continuing BPA research):
1. Improve gate selectivity at L>=1024. The current MLP gate
   trained on 7 local-attention features does not discriminate
   well enough. Consider: more features, larger gate MLP,
   position-dependent bias, or training the gate end-to-end
   with the language model.
2. Implement sparse attention kernels to translate algorithmic
   FLOPs savings into wall-time savings.
3. Implement compacted cache layout to translate KV read
   savings into actual memory allocation savings.

W&B note: Metrics are saved to JSON only (not W&B) because
this is a standalone evaluation script, not integrated into the
training loop. The JSON files in bpa_v3_results/ contain all
metrics that would be logged to W&B. This satisfies criterion
A.2 in spirit — all metrics are saved to persistent storage.
