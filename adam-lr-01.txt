# Step 1: Implement per-layer LR scaling (minimal + flag gated)
# ============================================================
# Core idea:
# - Adam already uses per-parameter preconditioning via v (second moment).
# - We add a per-layer LR multiplier derived from the layer’s aggregate v.
# - This is a coarse “layerwise natural-gradient-ish” adjustment.
#
# Requirements:
# - Must not affect baseline when flag is off.
# - Must work with standard AdamW in PyTorch.
# - Must handle tied embeddings / shared parameters sanely.
# - Must not break weight decay parameter grouping.
#
# Design:
# - Keep optimizer param groups as-is.
# - Apply LR multiplier via:
#     Option 1 (preferred, minimal): add per-group "lr" values updated on the fly.
#     Option 2: add separate param groups per layer (more intrusive).
#
# Preferred approach:
# - Build a mapping from parameter -> layer_id (string) using module names.
# - Define “layer” as transformer block i (e.g., h.0, h.1, ...), plus embeddings, plus lm_head.
# - During training, after optimizer has v estimates (needs at least 1 step),
#   periodically compute layer stats from optimizer.state[p]['exp_avg_sq'] (AdamW).
# - Compute lr_mult per layer, then apply to the param groups by setting group['lr'] = base_lr * lr_mult_for_that_group.
#
# Important: param groups may contain params across multiple layers (especially if grouped by decay/no_decay).
# So you need one of:
#   A) Split param groups by (decay_flag, layer_id) once at optimizer creation.
#      This is still reasonably small and deterministic.
#   B) Maintain group base_lr and a group layer_id tag.
#
# Implement A:
# - At optimizer setup, instead of two groups (decay/no_decay), create many:
#     (layer_id, decay/no_decay) -> param list
# - Store group metadata: {"layer_id": "...", "decay": bool, "base_lr": lr}
# - Scheduler should still scale group['lr'] from base_lr.
#   Easiest: maintain scheduler on a “global factor” and set group lr each step as:
#       group['lr'] = base_lr * sched_factor(step) * lr_mult[layer_id]
#   If existing scheduler directly mutates group['lr'], intercept by storing base_lr and recomputing after scheduler step.
#
# CLI flags to add:
#   --layer_lr_fim (bool, default false)
#   --layer_lr_fim_power {1,2} (default 1)
#   --layer_lr_fim_stat {median,mean,trimmed_mean} (default median)
#   --layer_lr_fim_eps (default 1e-12)
#   --layer_lr_fim_ref {median,mean} (default median)
#   --layer_lr_fim_clamp (default 4.0)   # r
#   --layer_lr_fim_update_every (default 50 steps)
#   --layer_lr_fim_warmup_steps (default 200)  # don’t apply until v is somewhat formed
#   --layer_lr_fim_log_jsonl path (default runs/adam-lr-01/layer_lr.jsonl)
#
# Computation details:
# - For each layer_id:
#     collect v = exp_avg_sq tensors for params in that layer (all params, both decay/no_decay ok)
#     compute stat_v = median(v.flatten()) (or mean)
#     s_l = (stat_v + eps)^(1/4)
# - s_ref is median of all s_l (or mean), computed on the same update tick
# - lr_mult = (s_ref / (s_l + eps))^power
# - clamp lr_mult to [1/r, r]
#
# Logging each update tick (stdout):
# - step, s_ref, min/max lr_mult
# - top 5 highest lr_mult layers + values
# - top 5 lowest lr_mult layers + values
#
