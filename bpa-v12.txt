BPA v12: Move from “works” to “frontier stress”.

Objective:
Establish whether BPA can become a true retrieval-sensitive, sublinear memory controller in a bandwidth-sensitive regime on a W7900.

This is not another tuning pass. This is a mechanism stress test.

GPU:
AMD W7900 48GB, ROCm. All runs GPU-only.

========================================================
PHASE 0 — HARD PRECONDITIONS
========================================================
0.1 Device hard requirement:
- Fail if torch.version.hip is None.
- Log:
  - torch version
  - ROCm version
  - device name
  - peak_gpu_alloc_mb/reserved
- No CPU headline runs.

0.2 Model requirement:
Use Qwen2.5-0.5B (max_ctx=32768) as baseline model.
Additionally attempt to load one larger model (>=1.5B) if memory allows.
- If larger model loads and runs L=8192 decode without OOM, include it in experiments.
- If not, continue with Qwen only.

========================================================
PHASE 1 — Force Bandwidth-Sensitive Regime
========================================================
Goal: Make KV traffic dominate latency.

1.1 Context sweep:
Run L in {1024, 2048, 4096, 8192, 16384} (all in-range).
Decode length >= 256 tokens.
Repeat each run 3 times.

1.2 Bandwidth stress regimes:
R1: batch=1 baseline
R2: microbatch=4
R3: microbatch=8 (if memory allows)

Log:
- p50/p95 decode latency
- tok/s
- KV_proxy bytes/token
- peak GPU memory
- % time in attention kernel (profile via torch.profiler)

Objective:
Identify L and regime where decode latency scales strongly with L (bandwidth-bound).

Acceptance:
At least one regime where doubling L increases latency >=1.6x.

========================================================
PHASE 2 — Replace Entropy with Retrieval-Grounded Signal
========================================================
Problem:
Entropy correlates with difficulty but not with retrieval necessity.

2.1 Implement attention-grounded supervision mode:
- During dense baseline run, record attention maps.
- For each token t:
  - Compute far_attention_mass(t) = sum of attention weights to tokens > W_min away.
  - Label token as "retrieval_required" if far_attention_mass > threshold (e.g., 0.2).

2.2 Train small controller (tiny MLP or logistic regression):
Input features:
- entropy
- residual norm
- logit margin
- layer-wise attention variance (cheap proxy)
Output:
- predict retrieval_required

Train on dense traces for small subset of sequences.

2.3 Replace heuristic pressure with learned retrieval predictor.
Controller:
- k_far(t) proportional to predicted probability.
- Maintain B_far_max cap.

Acceptance:
On held-out sequences:
- ROC-AUC > 0.75 for retrieval prediction.
- k_far spikes correlate with high far_attention_mass tokens.

========================================================
PHASE 3 — True Sublinear Scaling Test
========================================================
Measure effective kept tokens vs L.

For each L:
- Compute mean_kept_tokens(L) for matched-quality configs.
- Fit scaling exponent beta in:
    kept_tokens ~ L^beta

Only use IN-RANGE L values.

Objective:
Determine whether beta < 1 in any bandwidth-sensitive regime.

Branch outcomes:
- If beta < 0.85 consistently:
    strong sublinear evidence.
- If 0.85 <= beta <= 0.95:
    marginal sublinear.
- If beta ~1:
    local window dominates; need architectural shift.

========================================================
PHASE 4 — Layer-Adaptive W (structural experiment)
========================================================
Hypothesis:
Not all layers require same local window.

Implement:
- W_l_min and W_l_max per layer.
- Early layers small W, middle moderate, late larger.
- Compare against uniform W.

Tune under matched-quality constraints.

Objective:
See if per-layer W reduces kept tokens without hurting quality.

Acceptance:
Layer-adaptive W reduces kept_tokens >=10% vs uniform W at same ppl tolerance.

========================================================
PHASE 5 — 32K Context Stress
========================================================
Run L=32768 single-sequence decode.
Measure:
- latency
- KV size
- stability
- matched-quality feasibility at 1% and 3%

Goal:
Determine practical max context under BPA before collapse.

========================================================
PHASE 6 — Branch Tree Generation
========================================================
Generate bpa_v12_branch_tree.md summarizing outcomes.

Branches:

BRANCH A: Retrieval predictor works + beta < 0.85
- Next: refine controller, publish mechanism narrative.

BRANCH B: Retrieval predictor works but beta ~1
- Local window dominates; explore compressing W via learned recency decay.

BRANCH C: Retrieval predictor weak (AUC < 0.7)
- Features insufficient; need richer signals (multi-layer attention entropy).

BRANCH D: Latency flat despite KV reduction
- Still compute-bound; move to larger model or increase microbatch.

BRANCH E: 32K unstable or quality fails
- Explore KV quantization or paged KV.
