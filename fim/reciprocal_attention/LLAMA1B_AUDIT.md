## Design decisions

The 1B lane reuses the same harness as 150M (`llama150m_matched.py`)
with only JSON config changes. The harness was extended with:

- GQA support via `_repeat_kv()` for models with `num_kv_heads < num_heads`
- Checkpoint saving after training for downstream eval
- HellaSwag and LAMBADA evaluation via `--eval-checkpoint`
- Configurable model label in surgical selection output

FIM collection uses `candidate_keep_top_k_layers=5` (vs 3 for 150M)
to accommodate the deeper 22-layer architecture. Top-8 heads selected
globally, with top-4 derivation available.

## Scaling rationale

- Same effective batch size (128) as 150M lane for fair comparison
- LR reduced from 6e-4 to 3e-4 (standard for ~1B models)
- Warmup reduced from 2000 to 1000 steps (expect fewer total steps
  per hour due to larger model)
- GQA 8:1 is interesting for RA since 32 query heads give richer
  surgical selection options across layers
