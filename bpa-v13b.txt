Goal:
Evaluate three compression families in a unified harness on W7900 ROCm, under matched-quality decode at long context:
A) Proper low-rank KV (per-head, calibrated)
B) KV-cache quantization (per-layer calibrated)
C) KVSplice TRAINED to preserve attention/logits (distillation), used as long-term memory

Hard constraints:
- Model: Qwen2.5-0.5B (max_ctx 32768) (optional 1.5B only if stable)
- In-range only for headline: L ∈ {4096, 8192, 16384, 32768}
- Decode steps >= 256, seeds 0/1/2
- Regimes: r1 batch=1, r2 microbatch=4 (auto-downshift allowed but must be logged)
- Matched-quality PASS requires ALL seeds satisfy tol. Catastrophic FAIL if any seed > 3× ppl_dense.

Deliverables:
- results/v14b/* JSON logs for every run
- bpa_v14b_final_report.md
- bpa_v14b_scoreboard.json
- trained artifacts:
  - lowrank_calibration/*.pt (ranks + bases)
  - quant_calibration/*.json (scales/bit schedules)
  - kvsplice_trained/*.pt (splice module weights)

========================================================
PHASE 2 — A) Proper low-rank KV per head with adaptive rank
========================================================
A.1 Calibration data:
Use a fixed calibration prompt set (N=64 sequences) with lengths matching L bands.
Store only what you need (sampled Q/K/V).

A.2 Objective:
Choose ranks r_{layer,head} to minimize attention-logit distortion on calibration:
Approx: sample queries Q from decode steps and minimize ||QK^T - QKhat^T|| over sampled positions.
Enforce a global rank budget to achieve a target compression ratio (e.g., 1.5×, 2×).

A.3 Implementation:
Chunked U storage, fixed bases (S matrices) per head/layer for speed.
Measure overhead. If overhead dominates, reduce calibration complexity and increase chunk size.

========================================================
PHASE 3 — B) KV-cache quantization per layer (learned/calibrated)
========================================================
B.1 Implement at least:
- INT8 per-channel per-layer quant (scales/zero-points)
- NF4-like block quant (simulate if needed)

B.2 Calibration objective:
Minimize attention-logit distortion on calibration prompts (sample Q):
- error on QK^T is the primary target; V error secondary.

B.3 Per-layer schedule:
Choose bitwidth per layer under a global bytes budget (greedy search ok).
Produce a “bit schedule” that changes with L if needed, but prefer one schedule that works across L.

========================================================
PHASE 4 — C) KVSplice TRAINED (no exceptions)
========================================================
C.1 Define KVSplice module precisely:
- Compress far tokens (<= t - W_min) into M memory segments
- Produce K_mem, V_mem that are used in attention alongside the near full KV window
- No token dropping. Only representation substitution.

C.2 Teacher/Student:
Teacher = dense (full KV)
Student = near full KV + far replaced by KVSplice memory

C.3 Training objective (must implement):
Primary: attention-logit distillation (Δattention KL)
For sampled layers/heads and sampled decode steps:
- teacher logits Lt = QK^T / sqrt(d)
- student logits Ls = QK_mem^T / sqrt(d)
Minimize KL(softmax(Lt/τ) || softmax(Ls/τ)) with τ in {1,2}

Secondary (optional but recommended):
Match attention outputs:
||softmax(Lt)V - softmax(Ls)V_mem||^2

C.4 Training protocol:
- Freeze base model weights.
- Train ONLY KVSplice parameters.
- Curriculum on compression ratio:
  Start with M = T_far/2, then T_far/4, then T_far/8 (or equivalent)
- Train budget: 5k–20k steps (or until validation stabilizes)
- Use mixed precision safely; guard NaNs.
- Save checkpoints per stage.

C.5 Evaluation:
After training, evaluate kvsplice_trained backend under identical harness.
Sweep at least 3 compression settings (different M) to produce a real rate–distortion curve.

Hard pass criteria for kvsplice_trained:
- Must PASS tol=3% at L>=8192 in at least one regime
- Must reduce resident_kv_bytes_est >= 30% vs dense
- Must not regress p50 decode latency in the bandwidth-bound regime

If it cannot meet PASS after training:
- Report it as FAIL with diagnostics (distillation loss plateau, KL not decreasing, instability), no excuses.

Phase 0-1: COMPLETE
- backends/ framework: base.py (ABC), lowrank.py, quant.py, kvsplice_kl.py

- Bug fix: configure() was resetting calibrated=False on each call,

- Dense PPL ranges: 4.5-15.2 across seeds/L values.
- Dense p50: 14.8ms (4K) to 68.1ms (32K) -- bandwidth-bound confirmed.

Phase 2 (Low-rank KV): COMPLETE -- FAIL
- SVD analysis reveals K and V have fundamentally different spectra:
  K: 99.9% energy at rank=19 (highly compressible, RoPE concentrates)
  V: 99.9% energy at rank=63 (NOT compressible, high-dimensional content)
- Separate K/V ranks implemented. avg K_rank=31, V_rank=63.
- K-only mode gives identical results to full lowrank (V drives quality).
- Calibration at L=32K (full position range) improves over L=4K
  but still fails: 8-24% PPL degradation at L>=8K.
- Root cause: fixed SVD basis cannot capture position-dependent RoPE
  rotations. Would need position-aware projection = new attention.

Phase 3 (Quantization): COMPLETE -- PASS for INT8, FAIL for INT4
- INT8 per-channel symmetric: PASS@1% at ALL L values. Nearly lossless
  (<0.2% PPL delta). Clear winner.
- INT4 block (block_size=32): FAIL. 14-21% PPL degradation, 1 catastrophic
  at 32K. 4-bit range too small for Qwen2.5 KV dynamic range.
- Calibration assigned 0/24 layers to INT4 (all too sensitive).

Phase 4 (KVSplice Trained): COMPLETE -- FAIL
- Training script: scripts/train_kvsplice.py
- Trained seg=2/4/8 compressors, 5k-20k steps each.
- Training unstable: KL loss oscillates 10-1000x across layers because
  shallow vs deep layers have very different attention distributions.
- Eval results: dramatic latency improvement (35-61% faster at 32K)
  but catastrophic PPL degradation at all segment sizes.
- kvsplice_seg8 at 32K: p50=26ms (61% faster) but PPL 20-53 vs dense 8-15.
- Root cause: shared segment compressor cannot preserve position-dependent
  RoPE information. Per-layer would help but negates memory savings.

Phase 5: COMPLETE

- Report: bpa_v14b_final_report.md
- Winner: quant INT8 (only PASS method)
- Pareto frontier: exactly one point. No method achieves >2x compression
