rgsa-v19.txt — RGSA v19: Query-Conditional Importance and Dynamic Budget Gating
(Ralph Wiggum Loop Enabled)

Context / Ground Truth (Non-Negotiable)

We have conclusively established:

1. Adam exp_avg_sq (diagonal Fisher proxy) measures parameter update geometry,
   NOT runtime state importance. (v14–v16 confirmed)
2. Static, head-level budget reallocation (variance-weighted or inverted)
   is dominated by seed variance and hurts quality.
3. Uniform allocation is near-optimal among all static policies tested.
4. Head importance is NOT a static property. Any signal averaged across inputs
   collapses under noise.
5. Therefore: the remaining unexplored axis is *conditional importance*.

This means:
- Importance must be evaluated *per example, per position*.
- Budget must be gated dynamically, not pre-allocated statically.
- RGSA should behave like adaptive computation, not static sparsity.

v19 explicitly abandons static importance and studies **query-conditional signals**.

We are NOT trying to beat uniform by clever weighting.
We are testing whether *conditional necessity* exists at all.

Primary Research Question (v19)

Does there exist a **query-conditional signal** that predicts when far-context
attention for a given head materially affects the output distribution?

Formally, we seek signals correlated with:

ΔKL(l,h | x,t):
  KL divergence between:
    p_full(y | x₁…x_t)
    p_local(y | x₁…x_t) with head (l,h) restricted to local_window

If no such conditional signal exists → RGSA is fundamentally a regularizer only.
If such a signal exists → dynamic gating is the correct abstraction.

Non-Goals

- No static reallocation.
- No claims of inference speedups.
- No global importance scores.
- No kernel work.
- No new routing networks unless justified by evidence.

High-Level Strategy

We proceed in three layers:

Layer 1: Measure conditional impact (ground truth)
Layer 2: Measure cheap conditional predictors
Layer 3: Test dynamic gating policies using those predictors

All phases must support:
- Exact budget matching
- Multi-seed validation
- Matched tokens_seen
- Identical training configs

Phase 0 — Conditional Impact Oracle (Ground Truth)

Goal:
Measure ΔKL(l,h | x,t) directly, at runtime, without approximation.

Implementation Tasks:

1. Extend ImpactKLTracker to conditional mode:
   - Measure impact at specific (batch_idx, position t)
   - Restrict only head (l,h) AND only far-context
   - Keep local_window unchanged

2. Sampling strategy:
   - For each batch:
     - Randomly select K positions t (e.g. K=4)
     - Round-robin over heads (l,h)
   - Track EMA per head AND per position bucket

3. Store:
   impact_kl[l,h,t_bucket]
   where t_bucket ∈ {early, mid, late}

Sanity checks:
- ΔKL ≈ 0 at early positions
- ΔKL increases with t for some heads
- Distribution is heavy-tailed, not uniform

Stopping rule:
If ΔKL is uniformly ~0 across all heads and positions → stop v19.

Phase 1 — Candidate Conditional Signals (Cheap Predictors)

We test signals computable during forward pass, *before* paying far-context cost.

Candidate signals (implement all):

1. Local-only uncertainty
   - Compute entropy of logits using local_window attention only
   - Hypothesis: high uncertainty → far-context matters

2. Local vs far disagreement proxy
   - Run a cheap auxiliary head with local-only attention
   - Measure KL(local || current hidden state projection)
   - No second full forward allowed

3. Attention demand signals
   - Query-key norm ratio
   - Query norm spike relative to EMA
   - Attention score variance across heads

4. Retrieval pressure indicators
   - Fraction of attention mass trying to escape local_window
   - Top-k attention score gap at local boundary

For each signal S(x,t):
- Log S
- Log corresponding ΔKL(l,h | x,t)

Phase 2 — Correlation Analysis (The Make-or-Break Test)

For each signal S:

Compute:
- Spearman rank correlation with ΔKL
- Conditional ROC-AUC:
  Predict whether ΔKL > τ (τ = percentile threshold)
- Stability across seeds

Acceptance criteria:
- Correlation must be:
  - Consistent sign across seeds
  - Stronger than any static head metric
  - Non-trivial (ρ > 0.3 OR AUC > 0.65)

If no signal passes → conclude:
“No usable query-conditional predictor exists at this scale.”

Phase 3 — Dynamic Gating Policies

Only proceed if Phase 2 finds a viable signal.

Policies to test:

A) Uniform RGSA (control)
B) Conditional gate:
   - Enable far-context only when S(x,t) > threshold
C) Budget-capped conditional:
   - Rank heads by S(x,t)
   - Enable far-context for top-B heads per token
D) Hybrid:
   - Always allow a minimal uniform budget
   - Add conditional budget on top

Critical constraints:
- Exact total tokens/query matching
- No extra compute beyond conditional logic
- Deterministic gating for reproducibility

Metrics:
- Val PPL vs tokens_seen
- ΔPPL at fixed compute
- Variance across seeds
- Fraction of tokens triggering far-context

Stopping Rules

Immediately stop if:
- Conditional policies show sign-flips across seeds
- Gains vanish after matching tokens_seen
- Instability exceeds uniform baseline

Final Outcomes (v19 Must Conclude One)

Case 1:
No conditional signal correlates with ΔKL
→ RGSA is a regularizer only. Close the research line.

Case 2:
Conditional signal exists but gains are marginal
→ Publish negative-but-clarifying result.

Case 3:
Conditional gating improves frontier stably
→ Proceed to v20: learned lightweight gate.

Deliverables

- rgsa_v19_final_report.md
- Correlation tables and plots
- Policy comparison curves
- Explicit statement of what does NOT work
- Clear recommendation: continue or stop

Ralph Wiggum Loop Instructions

Repeat:
1. Implement minimal functionality
2. Run smallest viable experiment
3. Analyze
4. Decide to continue or stop

Do NOT:
- Add features without evidence
- Tune thresholds to rescue bad signals
- Rationalize noise

End State Definition

v19 is complete when we can answer, with evidence:

“Is head importance fundamentally conditional, or effectively uniform in practice?”

That answer — yes or no — is the result.
